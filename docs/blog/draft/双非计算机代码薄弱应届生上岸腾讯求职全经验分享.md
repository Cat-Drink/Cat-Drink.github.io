# 双非计算机代码薄弱应届生上岸腾讯求职全经验分享

## 个人背景

双非一本计算机专业大四，成绩中等偏上，代码能力薄弱：LeetCode 简单题经常卡顿，数据结构期末仅及格。
大三下秋招启动时内心焦虑，评估自身情况：考研上岸概率低、考公竞争激烈，决定直接就业。
看好 AI 行业赛道，计划依靠产品思维弥补代码短板，跟随 GitHub 项目、Vibe Coding 方式自制小项目，着手撰写简历求职。

## 投递整体情况

前期盲目海投效果极差，累计投递上百份简历，仅 12 份收到回复、4 次面试机会，最终成功拿到腾讯 offer。
初期两周投递零反馈，心态濒临崩溃，甚至考虑转去内卷研发岗刷题。
后期调整策略，稳定每日操作 \+ 精投定制简历，求职反馈率明显提升。

### 投递实操策略

1. 每日固定刷新招聘平台 BOSS，主动向 10\-15 个岗位打招呼；同步补充产品思维、AI 行业基础知识，保证面试有内容可聊。

2. 放弃大范围海投，转为精准投递：每投递一个岗位，单独修改简历与打招呼话术，对标 JD 里要求的技能点匹配自身经历，单份简历调整耗时约半小时，大幅提升 HR 回复率。

3. 意向公司 / 收到面试邀约前置调研法：

    - 完整浏览企业官网，下载公司主力产品深度体验；

    - 整理 2\-3 条产品优缺点、可落地优化建议；

    - 面试过程主动向面试官提出分析，极易给面试官留下深刻印象，本人第一份 offer 依靠该方法拿下。

## 简历撰写技巧

1. **求职意向必填**
清晰写明目标岗位（开发 / 产品等），不要让 HR 猜测求职方向；期望薪资适度抬高填写，HR 普遍会议价，填写过低企业会直接按低标准定薪。

2. **专升本学历写法**
简历教育经历仅标注本科院校即可，无需详细拆分专科阶段，参考网上标准模板，优先保证第一印象。

3. **专业技能填写**
在校所学相关课程全部罗列，前两条放置与应聘岗位高度匹配的技能，其余内容可补充凑数；
填写潜规则：接触了解过 = 会使用，熟练操作 = 精通，面试不会全部考察简历标注技能。

4. **项目经验包装**
校内参与过的项目均可撰写，仅辅助工作也可描述为深度参与；核心要求是能完整、逻辑通顺地讲清项目流程，适度优化包装经历。

5. **获奖情况美化**
参与过竞赛如实填写，无竞赛经历可合理撰写校级创新奖、院级优秀项目等奖项，自行命名即可；多数企业不会现场核验证书，属于简历加分项。

## 面试全流程注意事项

### 自我介绍规范

禁止照着简历原文朗读，使用三段式结构，时长控制在 1 分 30 秒以内：
过去做了什么 \+ 现在为什么应聘本岗位 \+ 入职后能为公司带来什么；
交谈时目光落在面试官眉心位置，兼顾自信与舒适，避免直视对视产生尴尬。

### 面试必问清楚 3 项核心权益

无需碍于情面，均属于求职者合法知情权：

1. 薪资核算：底薪税前 / 税后标准、绩效发放规则、是否存在年终奖；

2. 五险一金：社保公积金缴纳起始月份、缴费基数、公积金缴纳比例；

3. 工作节奏：加班频次、双休 / 大小周制度、是否发放加班费。

### 面试结尾反问环节技巧

HR 收尾询问「你还有什么问题吗」时，不能直接回答没有，会让面试官认为你对岗位缺乏兴趣；
提前准备 1\-2 个问题，示例：

1. 新人入职是否配备专属带教导师；

2. 该岗位日常以团队协作为主，还是独立完成工作居多；
通过提问同步判断岗位、团队是否适配自身需求。
