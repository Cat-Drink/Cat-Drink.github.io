# 怎么包装项目

最近有同学问我，项目是从GitHub上参考来的，或者是跟着别人的课程做出来的，这种项目怎么写进简历，才能显得更扎实一些、更像自己真正做过的东西。

首先，最重要的一点不是急着包装，而是你至少要对项目有比较高的掌握度，最好能达到80%左右。这里完全可以借助AI帮你提升理解效率，比如让AI先通读整个项目，帮你梳理核心模块、主要函数调用关系，以及整个项目的执行流程。这样你至少能先把项目的大框架和关键逻辑弄明白。

在这个基础上，你可以再把项目按白话的方式拆成五段。比如这个项目是做什么的、整体流程是什么、核心模块有哪些、你重点理解了哪些部分、遇到了什么问题以及怎么解决的。先用你自己的话把这五段内容写出来，再交给AI去帮你润色成更适合写进简历的表达。

比如提示词可以这样写：“我要面试xx岗位，请把下面这五段内容整理并润色成一份适合写在简历里的项目描述，包含项目名称、技术栈和项目要点，语言要真实自然，不要出现类似‘优化了80%’这种夸张表述。”

另外，我不太建议大家在简历里随意写一些特别精确、特别夸张的数据，比如“CPU占用率下降30%”、“故障率下降50%”之类。因为对大多数人来说，不管是自己做的项目还是工作经历，真要写这种非常具体的数据，往往很难自圆其说。面试官很可能会继续追问：这个数据是怎么测出来的，测试环境是什么，基线版本是什么，统计口径又是什么。很多人一追问就接不住了。

而且这种写法现在也很容易让人一眼看出来是AI味比较重的简历。相比之下，写得真实一点、具体一点、能讲清楚项目流程和你的实际参与内容，反而更有说服力。
