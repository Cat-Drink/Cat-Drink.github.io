# 计算机专业求职经验分享

## 作者：木木子

### 个人背景

我是二本计算机专业的学生，项目也不算特别硬核。虽然学校不错，但我的水平在班里真的排不上号，成绩在班里是中等偏下，够不上奖学金，也拿不出好的项目和实习。大四找工作的时候其实也很慌，因为身边有人拿大厂实习，有人算法刷了几百题，而我当时还是新兵蛋子。去年秋招海投了几千封简历，要么是已读不回，要么就是被拒绝。也是历经万难吧，还好最后一共拿了几个offer，想说说我的看法，仅供参考。

### 求职建议

#### 1. 机会与匹配度
机会肯定是有的，不是学历越高越好进，是你简历与职位描述的匹配度越高越容易得到面试机会。把这两方面准备好，进面机会还是蛮大的。校招一般看重的是可培养性，注重基础，把握好机会，进大厂的概率还是很大的。

#### 2. 找对适合的方向
不要一股脑的往那些技术要求特别高的岗位挤，选岗很重要。
- **普通开发岗**：比如Java后端、前端开发、测试开发，这类岗位需求量最大
- **软件测试或测试开发**：对算法要求低一些，但也有技术含量
- **运维、DevOps、实施工程师**：技术门槛相对友好
- **传统行业IT岗**：比如银行、国企、制造业的信息部门，工作稳定但节奏没互联网那么卷

#### 3. 公司选择策略
- **大公司**：卡学历卡技术栈
- **小公司**：更看重态度和学习能力
- **优先选择**：成立3-5年、业务稳定的小公司
- **入门岗位**：助理开发工程师、技术运维，这些岗位门槛低，是入门好跳板

#### 4. 投递技巧
- **一定要海投**：多投才能有机会，不要因为公司小就不投，小公司也能积累经验
- **关键词匹配**：投之前把岗位要求里的关键词圈出来，比如"会基础Java"、"了解Vue"，自我介绍里主动提这些点，让面试官知道你符合基本要求

#### 5. 投递时间策略
- **招聘高峰**：春招2-4月、秋招9-10月是高峰，错过就盯补录
- **最佳投递时间**：周二到周四投递回复率高，上午9-11点、下午2-4点是HR看简历的黄金时间

#### 6. 简历准备
普通学生也可以写出不错的简历，关键是项目：
- **项目数量**：建议至少准备2到3个完整项目
- **项目复杂度**：不一定要特别复杂，但要自己能讲清楚
- **项目类型**：比如一个简单的电商系统、博客系统、管理系统，这些都可以
- **项目描述**：一定要具体

#### 7. 面试准备
很多面试官并不指望你答出特别难的问题，更看重基础：
- **必问基础**：Java基础、数据库、HTTP、操作系统基础这些，基本是必问的
- **基础重要性**：把这些基础搞扎实，面试通过率会高很多

#### 8. 项目讲解准备
项目一定要提前准备好讲解逻辑：
- 项目背景是什么
- 用了哪些技术
- 你负责哪部分
- 遇到什么问题
- 怎么解决的

只要逻辑清晰，面试官一般不会太刁难。
