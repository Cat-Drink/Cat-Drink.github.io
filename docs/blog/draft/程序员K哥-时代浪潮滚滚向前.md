# 程序员K哥：时代浪潮滚滚向前

周末和几个在大厂的学员朋友都聊了聊。我和大家说说后续的方向。

以后程序员会演变的最终形态，就是AI全栈业务专家，中间可能会适当过渡，也就是分为AI开发全栈和AI业务专家。

第一种大家其实好理解，很多大厂已经在陆续推进了，前后端客户端不分家，AI一把梭，不但强调技术能力强，而且还要求效率高不能有明显短板。

第二种是往后10年全人类社会最稀缺的人才，我们也正在逐步培养大家的能力和意识。AI业务专家。

大家以后不必再问前端要不要学后端要不要学 Java要不要学 Python要不要学，说实话这些语言都只是工具，未来只要是工具的应用和实操层面，在AI的加持下学习成本基本可以忽略不计。

那么公司重点关注什么能力？大模型薪资是不是真的这么高？什么能力未来在市场上会具备高稀缺和高溢价属性？

我创业这么多年，从老板的视角给你们解读一下为什么现在大家对AI这么焦虑。薪资为什么这么高。

不止是打工人，其实现在的老板和大厂远比普通人焦虑的多。

再说Agent开发，其实市面上对于Agent开发也并没有清晰统一的标准，这也是一件好事，混乱就是阶梯，乱世出英雄，在一个行业没有清晰标准的时候才是蓬勃发展的时候，也是普通人更好入局和浑水摸鱼的时候。

没有标准是好事，有些人可能会好奇这么乱为什么是好事？因为当一切都建立标准的时候，也就是开始论资排辈，勾心斗角的时候，这个时候行业的竞争会转变为个人vs个人，组织vs组织之间的行业壁垒，社会资源的争夺，这种形态的竞争，会打压创新，抬高入门门槛，往往新人不具备任何优势。

参考你们现在看到的互联网，java，金融领域，都是这种程度的竞争。大家都在进行无意义的八股文内卷，学历歧视，工作年限和稳定性横向对比，本质上都是行业内卷的标志。

而在一个新兴行业里面，是不存在这些问题的。

那到底什么是Agent开发？接下来我的定义可能会更加广义。大家理解的Agent开发核心竞争力可能就是学一堆LLM，MCP，RAG技术名词，再结合一堆莫须有的业务场景和自己YY出来的解决方案就觉得行了，跟进一步就是深入理解XXX原理，这是传统的技术导向思维。

我首先强调一遍，技术是最不值钱的东西，这句话我在几年前就已经说过，技术是入场券，一场盛大，集齐全社会目光和决定全人类社会共同命运的AI大秀场，只会买票入局的人，是永远站不到舞台中央吃到真正行业红利的。

Agent开发的本质其实就是利用AI通用大模型的能力，加上MCP，Skill，RAG构建新的业务系统和工作范式。

和传统移动互联网其实没有区别。那么以前的移动互联网在干嘛？说白了就是把以前excel里面的数据搬到数据库里面。再对这一堆数据写一堆屎山代码做增删改查。

新兴技术行业的发展分为几个阶段：
1. 利用工具对原来的行业降本增效
2. 国家和资本打造新兴技术生态和社会基建
3. 开创新的商业模式，形成平台巨头，拿到大量的B端或C端用户，再利用资本积累起来的海量用户，数据，团队的壁垒形成事实意义上的垄断。

什么叫降本增效？比如之前做个hao123，网站聚合产品就能拿到融资，普通人只要会建站搬运信息，就能拿到一年几十上百万的广告费。

你把各种员工，财务日常的excel信息搬到数据库里面，就变成了各种erp，mes系统。

什么叫商业模式？把司机和乘客的信息搬到mysql，就变成了滴滴，把商家和食客的信息搬到数据库，就变成了美团。就算是字节头条，早期也是对各大互联网平台的首页新闻做各种爬虫搬运分发才起家的。

什么叫行业基建？移动互联网时代的基建是智能手机和4G网络的推广，没有人手一台智能手机就不会有app生态，没有4G网络和流量费的下降，你只能在网络上用家里的大头电脑或者自己的大哥大手机在qq农场偷菜，刷抖音打游戏是想都别想了。

你说这些技术难不难？其实技术都不难，难的是创新的idea和对行业和社会的深刻洞察。

有人说K哥你既然说技术简单，那为什么没有人再做出第二个美团，滴滴？

原因很简单，因为这种覆盖衣食住行平台型的互联网公司大家基本只需要一个，最多两三个偶尔竞争，一旦先发优势形成行业壁垒，其他人再想进来就需要花天价去培养用户习惯和建设生态，就算有人有这么多的钱看好而且愿意砸向这个行业，那他为什么不直接买这家巨头的股票呢？

Agent才是未来5-10年真正意义上属于所有普通人的时代风口，而不是AI。

以前所有人都需要有智能手机，需要地方都需要网络。以后所有人和公司都需要有自己的AI跟班，Agent系统。

AI生态的建设属于大厂，资本，比如万卡集群，通用AI大模型的训练。

但是Agent的建设属于每一个普通人。而且这是通用大模型和所有大厂无法解决的问题，因为大厂只能解决平台和基础建设以及生态的问题，但是大厂无法深入所有行业，无法了解所有行业的痛点和需求。

现有的AI架构是绝对不可能形成AIGC的，也就是大家说的通用人工智能，也就是说一个AI全能神，他能帮你解决所有问题，完成所有关键的人生动作和决策。这是不可能的。

因为Transformer本质上就是大模型概率预测，甚至可以简单理解为大号的搜索引擎。

只要是概率就一定会犯错，大模型最多能在通用能力上提高到70分，那如何让一个70分的实习生，把工作做到90分？甚至120分？

这就需要Agent，通过市面成熟的通用大模型能力，结合自己的行业理解和各种技术，对AI的输入输出做各种限制和存储，你想让AI有更好的记忆能力，那就分为短期记忆，长期记忆，对话记忆，和工作记忆。

你想让AI除了有自己的脑子还能长出自己的手脚，那就需要MCP，各种层出不穷的Skill，Open Claw。

你想让AI在某个垂直领域有独到的见解，并且把这个领域实践专家的经验复制给他，那就需要不断训练本地知识库甚至私有大模型。甚至通过结构化的提示词对AI进行约束训练，也属于Agent开发工作中的一部分。

通用大模型token太贵，响应太慢，不同场景适配度不够，那么就需要多模态，各种小模型的适配和兼容。

这里每一项技术单独抽出来都不难，难的是什么？难的是行业深度和工程化落地的细节。

做一个demo很容易，做出一个成熟被市场和用户接受的产品，很难。

有几个建议，所以希望大家在未来的学习和求职中牢记在心：

1. 终身学习，不然一定会被飞速发展的时代淘汰
2. AI即未来，不要害怕和恐惧，要拥抱他
3. 不要对自己定义为 java+ai，python+ai，只要是公开技术你都能学到，但是行业特殊的理解和实践经验以及技术+业务的解决方案，要么选择接受培训，有老师教，要么只有以身入局，在具体的公司和行业里面，才能学到让自己真正成长的东西
4. 把自己定义为AI赋能行业，AI解决方案的工程师，AI技术专家，一定是AI理解在先，解决方案在先，降本增效在先，最后才是所谓的技术。

5. 出去找工作，更不要害怕，相信我，大部分老板和面试官也是傻der，他们也是活在AI恐惧下的普通人。能把我刚才的观点牢记在心，并且理清各个技术栈的生态作用关系，把老师阐述的技术点和解决方案讲清楚，出去面试就足矣吊打很多半桶水了。

在这个混乱的时代，你只要比普通人领先一点点，就超越了绝大部份人，因为在这个群之外，绝大部分人都是普通人。

不管黑猫白猫，能抓到老鼠就是好猫。同理在老板眼里，不管什么技术什么方案，能解决问题，降本增效就是高薪人才。

传统移动互联网的时代已经过去了，Java前端后端这些职业未来都不会消失，但是行业极度内卷，岗位会急剧压缩，普通人已经失去在这种赛道里面内卷的资格。
