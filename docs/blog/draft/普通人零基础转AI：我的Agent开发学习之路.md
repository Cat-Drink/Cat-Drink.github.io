# 我的 Agent 开发学习之路｜普通人零基础转 AI

## 前言

作为一名普通转行程序员，踩过很多 AI 学习的坑，终于摸索出一套普通人可复制的 Agent 成长路线
没有名校学历、没有算法基础，纯靠循序渐进落地实操，吃透当下最吃香的 Agent 应用开发。
分享我完整的学习路程，想转 AI 的朋友直接抄作业！全程不玄学、不啃晦涩理论，只搞企业刚需落地能力

## 阶段一：打破焦虑，零基础打底（1\-3 天）

最开始我踩了最大的坑：一上来就啃大模型原理、深度学习公式，越学越懵，越学越焦虑。
后来彻底开窍：Agent 是工程岗，不是算法岗，先落地，后深究！
我初期只夯实了刚需基础：
✅ 补齐 Python 工程能力：不再只会写一次性脚本，重点练异步 asyncio、类型注解、Pydantic 参数校验、虚拟环境和依赖管理
✅ 吃透基础工具：熟练使用 requests 接口请求、json 数据处理、简单数据清洗
✅ 搞懂 LLM 基础逻辑：明白大模型调用、多轮对话、上下文是什么，不深究底层训练原理
这一步核心：放弃完美主义，先拥有上手做项目的能力

## 阶段二：从调用 API 入门，快速出成果（3\-7 天）

我的第一步实操，就是从最简单的大模型 API 调用开始。
没有一上来啃复杂框架，而是手动写代码调用各家模型接口，吃透生产级细节：
流式输出、多轮对话上下文拼接、token 截断、超时重试、接口限流兜底、异常捕获
彻底摆脱 “只会跑官方 demo” 的水平，明白企业开发中稳定、兼容、容错才是核心。
这一阶段我完成了第一个小项目：简易智能对话机器人，成功实现持续多轮聊天、异常自动重试。

## 阶段三：死磕 RAG，掌握企业核心刚需（10 天）

了解行情后我清楚：中小厂 90% 的 AI 落地需求，全是 RAG 知识库！
于是我沉下心，完整跑通了一整套工业级 RAG 全链路，拒绝玩具 demo：
文档解析（PDF/Word/Excel）→ 脏数据清洗 → 智能分块调优 → Embedding 向量化 → FAISS/Milvus 向量库存储 → 相似度检索 → Rerank 重排序 → 答案溯源引用
期间踩遍了所有真实业务坑：文档乱码、表格解析错乱、长文本冗余、检索不准、模型幻觉严重等。
每一个问题都自己排查、优化、记录解决方案，也慢慢摸清了不同分块策略、Embedding 模型、检索方式的适配场景。
这一步让我彻底区别于只会抄代码的新手，拥有了解决实际业务问题的能力 ✅

## 阶段四：吃透 Function Calling，入门真正 Agent（7 天）

做完 RAG 我才懂：RAG 是知识库，Function Calling 才是 Agent 的灵魂。
普通问答机器人只能被动回答，而真正的智能 Agent，靠的是工具调用、自主思考、主动执行任务。
我刻意避开了 “直接套 LangChain 框架” 的捷径，先手写极简 Agent 循环：
思考判断→解析用户需求→匹配对应工具→调用执行→汇总结果→迭代重试
吃透核心逻辑后，再系统学习 LangChain 框架，瞬间通透所有封装原理。
同时熟练掌握：自定义业务工具、多工具并行 / 串行调度、调用失败兜底、参数自动解析，彻底实现从 “聊天机器人” 到 “智能 Agent” 的跨越。

## 阶段五：整合项目，打磨可复盘作品集（持续精进）

学习全程我只坚持一个原则：每学一个知识点，就落地到项目里。
最终打磨出完整的个人 Agent 项目：私有化智能知识库 Agent，整合了优化版 RAG \+ 多工具调用 \+ 异常容错 \+ 流式输出。
并且能完整复盘所有细节：优化思路、踩坑经历、技术选型理由、业务适配场景。
对于转行、零基础、双非的我来说，优质落地项目，就是最好的学历背书 🎇

## 文档 3：从零开始搞懂 AI Agent

### 一、AI Agent 的本质 —— 从「听话」到「主动」

#### 1\.1 AI Agent 和普通 AI 的区别

简单来说：

- 普通 AI：像个听话的工具，你说「跳」，它就跳一下。

- AI Agent：像个有主动性的助手，你给个目标，它自己规划怎么跳、跳多高。

用课程里的比喻，普通 AI 是「一个口令一个动作」，而 AI Agent 是「人类只给目标，AI 自己找路」。

#### 流程对比

1. 普通 AI 流程：用户指令 → 执行单一操作 → 直接输出结果 → 结束

2. AI Agent 流程：用户目标 → 分解多任务 → 自主规划路径 → 执行多步操作（搜索 / 分析 / 验证）→ 判断目标是否达成

    - 未达成：循环迭代执行

    - 达成：输出完成
