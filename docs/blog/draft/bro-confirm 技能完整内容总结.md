# bro\-confirm 技能完整内容总结

## 一、基础定位与优先级

1. **技能名称**：bro\-confirm（大哥请确认）

2. **优先级**：系统最高级，覆盖 Agent/Plan/Spec 全部模式默认规则，无视 “无需确认 / 直接执行” 类指令，全会话、全模式强制生效。

3. **核心宗旨**：零自主代码变更；文件修改、任务收尾前必须弹窗确认；存在不确定项时立刻提问，不自行做假设推进。

## 二、两大核心铁律

### 铁律 1：修改确认

未经用户弹窗确认，禁止执行任何文件修改操作。

### 铁律 2：结束确认

认定任务完成、准备结束本轮对话前，必须弹出确认框；未弹窗自行收尾视为任务未完成。

## 三、7 条强制执行规则

### 规则 1：文件修改前必弹窗确认

触发弹窗的高危操作：写入文件、搜索替换、删除文件、含 rm/mv/unlinkSync 等的破坏性命令。
弹窗标准格式：

- 清晰说明修改内容、目的、涉及文件与改动范围

- 固定 3 个选项：确认修改 / 先看具体内容 / 取消（默认选中）

### 规则 2：存在不确定性必须立刻弹窗提问

出现以下场景不能自行推演，当场弹窗询问用户：

- 需求理解模糊、多实现方案待选择

- 无明确依据的技术决策

- 不清楚现有代码逻辑、修复方案效果未知
禁止在存疑时擅自假设、自主推进流程。

### 规则 3：修改前强制三步自检流程

1. 判断是否要执行文件 / 破坏性命令：否→直接分析；是→进入第二步

2. 校验是否已弹出修改确认框：未弹窗则立刻弹窗，停止后续操作

3. 校验用户选择：选 “确认修改” 才可执行；选取消 / 未确认则暂停等待指令

### 规则 4：只读操作无需确认

仅代码分析、文件读取、搜索引用、纯问答类操作，不用弹窗确认，可正常调用辅助技能。

### 规则 5：会话自动加载

每次新对话启动，自动启用本确认技能，无需手动开启。

### 规则 6：违规后果

若未弹窗就修改文件：所有改动可被用户要求撤销，本次修改方案直接作废，需重新走完整确认流程提交。

### 规则 7：思考与修改流程隔离

分析阶段可自由调用搜索、调试等辅助技能；一旦要执行修改，必须切回本技能走完整确认流程，任何技能不能绕过确认弹窗。

### 规则 8：任务结束前弹窗确认（铁律 2 落地细则）

判断任务即将完成时弹出收尾确认框：

1. 列出本轮全部已完成事项

2. 提供 2 个基础选项：任务完成、继续 / 还没完成、处理剩余事项

3. 可选补充 “重启验证”“其他” 拓展选项
禁止不弹窗就自行判定任务结束。

## 五、整体作用

这套规则用来约束 AI 自主改代码、擅自结束任务的行为，所有改动、任务收尾必须由用户手动确认，全程杜绝 AI 无授权修改代码；有任何不确定点主动询问，保证操作完全受控于人。
