## 01 frontend\-design：产出具备专业质感的产品级网页

### 核心定位

面向高质量前端界面设计与实现工作流，解决 AI 生成页面廉价模板感问题，产出接近商用产品的网页。

### 1\. 解决的痛点

1. AI 原生生成页面模板拼接痕迹重，视觉杂乱廉价

2. 布局、信息层级、留白、整体气质把控差

3. 页面能运行，但缺乏成熟产品质感

### 2\. 完整工作流程

1. 审美定调：先确认整体视觉风格、品牌气质方向

2. 层级规划：梳理页面信息架构与内容优先级

3. 页面实现：搭建组件体系，完成页面代码开发

4. 质检校准：统一校验视觉规范、版式一致性

### 3\. 适用业务场景

落地页（Landing Page）、SaaS 数据仪表盘、作品集页面、移动端 App 网页 UI

### 4\. 交付价值

- 页面风格统一，摆脱 AI 拼接廉价感，贴近商用产品

- 稳定的版式、留白、视觉层级判断能力

- 能力升级：从 “能写出可运行前端代码”→“能做出有专业审美的产品界面”

### 5\. 一句话总结

让 coding agent 从 “只会写页面” 升级为 “能制作具备成熟审美质感的产品界面”。
**关键词**：UI 质量、前端版式、视觉一致性、产品级界面

2. 大量日志信息干扰，难以快速锁定代码缺陷位置

3. 常规 AI 仅能生成代码，不具备解读工程 CI 报错、闭环修复能力

### 2\. 完整工作流程

1. 日志读取：拉取 GitHub Actions 完整 CI 运行日志

2. 故障归纳：总结失败类型、报错上下文、影响代码范围

3. 修复开发：定位代码根因，生成对应修复代码

4. 验证闭环：重新模拟 CI 校验，直至流水线全部通过

### 3\. 适用业务场景

单元测试失败、Lint 代码规范报错、项目构建打包失败、TS 类型检查报错

### 4\. 交付价值

- 快速穿透海量日志直达问题根源，省去人工排查时间

- AI 深度参与工程修复闭环，不只是单纯分析报错

- 能力升级：从 “能修改代码”→“能读懂 CI、自主修复流水线故障”

### 5\. 一句话总结

### 2\. 完整工作流程

3. 代码落地：针对性修改代码，关联对应评论条目

4. 结果汇总：生成每条评论的处理摘要、统一回复文案

### 3\. 适用业务场景

### 4\. 交付价值

- 高度贴近真实团队工程协作模式，而非单次孤立代码生成

- 避免评审意见遗漏，减少来回沟通成本，加速 PR 合并

- 能力升级：从 “只会提交代码”→“能完整处理 PR 评审全流程”

### 5\. 一句话总结

让 coding agent 像团队正式工程师一样，完整处理 PR 评审评论并推进代码合并闭环。
