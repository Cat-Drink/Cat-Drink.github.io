# 来自 27 届计算机男的字节 AI Agent 面试复盘

27 届计算机科学的，刚结束字节 AI Agent 岗的三面。说实话心里还是很忐忑的，虽然我也拿过一些中小厂的 offer，字节这种大厂能不能过还是有点没谱。

## 面试整体感受

面之前以为 Agent 面试就是聊聊八股、聊聊项目。结果真进去才发现，其实跟大家想象的也不太一样。字节面下来整体感受就是：

- **一面**看你基础到底扎不扎实；
- **二面**疯狂深挖项目；
- **三面**直接开始聊 Agent 的系统设计和业务落地。

整个面试过程压迫感其实是挺强的。我只能说面试官真的很懂怎么压力人，但又不是那种故意 PUA 你的压力面。很多问题都不是网上能背到答案的，而是顺着你的项目一点点往下刨。

我原本以为自己准备的已经非常充分了，结果还是有几个问题被问懵了。

---

## 被问懵的几个瞬间

### 多 Agent 协作项目的深挖

我在简历里写了一个**多 Agent 协作**项目。本来我以为会问用了什么框架、为什么选它。

结果面试官问的是：**如果其中一个 Agent 持续输出低质量结果，你是怎么动态调整整个工作流的？**

我当时脑子直接宕机了 2 秒，后知后觉地发现是在问我有没有真正理解 Agent 的系统。后面还继续追问了：**上下文怎么压缩的？长对话怎么防止 token 爆炸？**

说实话如果不是自己亲手做的，这里根本顶不住。

### 面试官不信你的简历

然后我发现一个特别真实的事情：这个面试官他好像不太信我简历里做的项目。他把考察重点都放在了这些设计到底是不是你自己思考出来的。

比如我有个 **RAG 项目**，面试官没有问我什么是向量数据库，而是直接问：**如果召回结果质量越来越差，你会先排查哪一层？**

其实这个问题一下子就把真正做过和只会调 API 的人区分开了。因为你真正跑过数据的人就知道，问题可能出在分块策略、嵌入模型、检索方式等等这些环节上。

---

## 工程化思维的重要性

然后还有一点我建议大家一定注意：**工程化思维一定要有。**

我记得当时面试官问我：如果让你做一个**抖音创作者 AI 助手**，你会怎么设计 Agent 的框架？

这个问题其实特别开放。我本来想从技术栈来讲，结果面试官直接打断了，跟我说：**先别聊框架，先聊业务流程。**

那一瞬间我突然意识到，其实很多人学 Agent 的只是停留在技术视角。而现在的 AI 团队其实缺的是真正把产品落地的人。

---

## 给 27 届冲 AI Agent 方向的几条建议

如果你也是 27 届，想冲 AI Agent 的方向，我感觉有几件事情特别重要：

### 1. 有真正跑通的 Agent 项目

最好有自己完整做过 Agent 的项目。别只会跑 demo，至少得能调用工具、有 memory，能真正跑通业务流程。

### 2. 把项目细节吃透

现在面试官特别喜欢顺着一个点无限往下问，尤其是：

- 为什么这么设计？
- 为什么不用别的方法？
- 有没有踩过坑？

很多人其实不是挂在不会，而是挂在**答得太虚**。

### 3. 多看真实业务场景

比如怎么做**客服 Agent**、怎么做**代码 Agent**、怎么做**创作者助手**。重点不只是技术，而是你有没有**工程落地思维**。
