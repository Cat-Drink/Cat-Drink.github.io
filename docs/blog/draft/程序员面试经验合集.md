## 1. 腾讯后端岗面试

### 整体感受
腾讯后端面试非常硬核，极其看重底层原理、计算机基础（系统级）以及大规模分布式系统的架构设计能力。八股文不仅要会背，更要懂底层源码和实际应用场景。

### 核心考点

#### 架构与分布式
- 说一下你的架构设计，如果是高并发高可用的架构，该怎么设计？
- Redis 缓存和 MySQL 数据库如何保证一致性？具体有哪些方案？
- 分布式锁的实现方案有哪些？你更倾向于哪种？为什么？
- 如果 Redis 缓存发生穿透、击穿、雪崩，该怎么处理？
- 微服务架构中，服务注册与发现的原理是什么？Consul / Nacos / Eureka 的区别？
- 分布式事务的解决方案有哪些（2PC、3PC、TCC、本地消息表、Seata）？
- 消息队列（如 Kafka、RabbitMQ）如何保证消息不丢失？如何处理重复消费？
- 什么是全链路追踪？它的核心原理是什么？

#### JVM 与 Java
- 了解哪些垃圾回收器？CMS 和 G1 的区别是什么？
- 说一下 JVM 的内存区域划分。哪些是线程私有的？哪些是线程共享的？
- 类加载机制的过程是怎样的？什么是双亲委派模型？为什么要设计双亲委派？
- 强引用、软引用、弱引用、虚引用的区别和应用场景是什么？
- 说一下你常用的 JVM 调优参数有哪些？发生 OOM 怎么排查？

#### 计算机网络
- 输入一个 URL 到页面加载出来的全过程是怎样的？
- TCP 三次握手和四次挥手的流程是怎样的？为什么挥手需要四次？
- TCP 是如何保证可靠传输的？（滑动窗口、拥塞控制等）
- HTTPS 的加密机制是怎样的？对称加密和非对称加密分别用在什么阶段？
- HTTP/1.1、HTTP/2.0 和 HTTP/3.0 有什么区别？

#### 算法与数据结构
- 说一下常用的排序算法，快排和归并排序的时间复杂度和空间复杂度是多少？
- **手写算法题**：最长回文子串（要求给出最优解法，并解释时间复杂度和空间复杂度）
- 反转链表（手写代码）
- LRU 缓存淘汰算法的设计思路是什么？如果要你实现，你会用什么数据结构？
- 动态规划和贪心算法的区别是什么？

#### Linux 网络编程与内核
- `epoll` 与 `select/poll` 的本质区别
- `epoll` 的底层实现（红黑树 + 双向链表），触发模式：水平触发（LT）与边缘触发（ET）
- 零拷贝技术（Zero-Copy）：`mmap`、`sendfile` 的工作原理

#### 高并发架构设计
- 海量数据存储与高并发读取场景下的分库分表策略（垂直拆分、水平拆分、Hash取模、Range范围划分）
- 分库分表带来的分布式ID生成问题（雪花算法 Snowflake、UUID、数据库自增步长、Redis分布式ID）以及跨库Join的解决方案

#### 项目相关
- 说说你在项目中遇到的最大的技术挑战是什么？你是怎么解决的？

## 3. 核心技术专题：滑动窗口
**来源**：程序员小星

### 1. 核心定义与作用
**滑动窗口** 是一种常用于算法优化（如双指针法解决子串/子数组问题）以及网络流控（如TCP流量控制）的核心机制。

### 2. 面试高频考察方向
- **算法层面**：
  - 如何在 $O(n)$ 时间复杂度内求出满足条件的最长/最短子数组、子串
  - 经典题目：无重复字符的最长子串、最小覆盖子串、长度最小的子数组
- **工程与架构层面**：
  - **微服务限流机制**：滑动窗口限流算法（对比固定窗口限流，它如何平滑突发流量、消除临界窗口的流量两倍冲击问题）
  - **TCP协议中的滑动窗口**：
    - 接收端与发送端的窗口同步机制
    - 流量控制（Flow Control）与拥塞控制（Congestion Control：慢启动、拥塞避免、快重传、快恢复）的区别与联系

## 4. Java 后端面试真题精选
**来源**：-搁浅_-

### 1. Java 基础与集合框架
- **HashMap 源码级深挖**：
  - HashMap 的底层数据结构是什么？（数组 + 链表 + 红黑树）
  - 为什么在JDK 1.8中引入红黑树？触发链表转红黑树的条件是什么？（链表长度大于8且数组长度大于等于64）
  - 为什么红黑树在节点数变少时（小于等于6）会退化为链表？
  - 负载因子（Load Factor）为什么默认是 0.75？
  - 扩容机制（Resize）的实现原理，JDK 1.7 的头插法导致死循环的原因，以及 JDK 1.8 的尾插法优化

### 2. 线程池与并发编程
- **线程池核心参数**：核心线程数、最大线程数、空闲线程存活时间、时间单位、任务队列、线程工厂、拒绝策略
- **拒绝策略**：
  - `AbortPolicy`（抛出异常，默认）
  - `CallerRunsPolicy`（调用者运行，谁提交谁执行）
  - `DiscardPolicy`（直接丢弃）
  - `DiscardOldestPolicy`（丢弃队列中最老的任务）
- **工作流程**：核心线程 -&gt; 阻塞队列 -&gt; 非核心线程 -&gt; 拒绝策略

### 3. JVM（Java虚拟机）
- **内存区域划分**：堆、方法区（元空间）、虚拟机栈、本地方法栈、程序计数器。哪些是线程共享的？哪些是线程私有的？
- **垃圾回收（GC）机制**：
  - 如何判断对象是否可以回收？（引用计数法、可达性分析算法）
  - GC 垃圾回收算法：标记-清除、标记-复制、标记-整理
  - 常见的垃圾收集器（CMS、G1）的特点和工作原理

## 5. 某大厂后端开发多轮面经
**来源**：-怎么又饿了-

### 【第一阶段：基础扎实度考察】
- **计算机网络**：
  - TCP 建立连接的三次握手与断开连接的四次挥手详细状态机变化（SYN, ACK, FIN, TIME_WAIT）
  - 为什么需要 TIME_WAIT 状态？为什么是 2MSL？
  - HTTPS 的加密原理：对称加密、非对称加密、CA证书、数字签名以及 SSL/TLS 握手全过程
- **操作系统**：
  - 进程与线程的区别；线程间通信与进程间通信（IPC）的方式
  - 什么是虚拟内存？分页与分段管理机制，页面置换算法（LRU, FIFO）
  - 什么是死锁？死锁产生的四个必要条件是什么？如何避免死锁？
  - 进程、线程和协程的区别是什么？
  - 并发和并行的区别是什么？

### 【第二阶段：分布式与中间件架构】
- **Redis 高并发缓存**：
  - Redis 的数据类型及其底层编码（SDS、ZipList、SkipList、HashTable）
  - 缓存三剑客：缓存雪崩、缓存击穿、缓存穿透的定义、发生场景及彻底解决方案（布隆过滤器、热点数据永不过期、随机过期时间）
  - Redis 持久化机制：RDB 与 AOF 的优缺点及混合持久化
- **分布式事务与一致性**：
  - CAP定理与BASE理论的深层含义
  - 常见的分布式事务解决方案（2PC、3PC、TCC、本地消息表、RocketMQ事务消息）

### 【第三阶段：综合场景设计与面试官对抗】
- **架构设计题**：如何设计一个秒杀系统？
  - 如何防止超卖（Redis预扣减库存 + Lua脚本原子性）
  - 流量削峰（引入消息队列 RocketMQ/RabbitMQ）
  - 静态化与CDN加速、多级缓存架构、恶意请求拦截（限流防刷）
- **面试总结**：面试官非常注重底层原理的连贯性，从一个基础 API 能一路问到操作系统内核或底层数据结构的内存布局，必须具备系统化的知识网络
