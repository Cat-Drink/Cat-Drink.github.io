# Agent 面试复盘：项目场景、技术选型与工程化深度

下半年到现在，我以面试官身份面了三十多个 Agent 方向的候选人。

那我自己也去面了几家做 Agent 的产品公司。看下来有个感受非常强烈：写自己的 demo 项目或过往项目，差别非常大。

---

## 面试官看简历的前 30 秒

面试官看简历的前 30 秒，会决定接下来追问什么东西。

他看的首先是你的场景适不适合做 Agent。比如说有些候选人简历罗列了 LangGraph、RAG、微调等，很全。

我问他：一个场景用豆包能不能解决？

他说能，但是我想做复杂一点。那这个就基本上也没有什么可问的。

为什么？你的场景如果用豆包就能解决，那你对现有技术改变不了，它是一个**伪需求**。那基本上可能就不会追问技术深度了，他会追问你为什么选这个场景，你答不来就悬了。

### 好的场景是什么？

好的场景是需要**多步推理**、**跨多个数据源整合信息**、需要**处理异步任务**——这种一次对话搞不定的东西，面试官才会觉得你这个 Agent 是值得深挖的。

---

## 技术方案：不是越多越好

你如果过了简历筛选，第一轮技术面试，面试官都会问你每个技术选型为什么选它。

最常见的问题就是技术方案写太满了。有些人写 LangGraph、Agent、Neo4j、Milvus、Chroma、RAG、LoRA 微调，七八个列在一起。

问他：你的数据量有多大？为什么用 Milvus？

他说数据量大概几千条——那就减分了。

为什么？Milvus 是分布式的生产级**向量数据库**，设计目标是千万级向量、高 QPS。几千条数据用 Milvus，技术选型没有考虑场景。

面试官不怕技术少，但是你要写清楚为什么选它。不然你连选它是为什么都不知道，那你怎么去用它呢？

## Agent 核心概念深挖

技术选型聊完之后，面试官一般会挑几个 Agent 的核心点深挖。

### 1. Workflow 和 Multi-Agent 的区别

有些人说加个 Router 分发任务就是 Multi-Agent——不是的。

Agent 之间要**自主决策**、能**相互调用**、能**通信**才算 Multi-Agent。固定流程分发汇总在一起，那只能叫工作流。

Anthropic 那篇《Building Effective Agents》出来之后，这个定义基本是行业共识了。所以说你要清楚。

### 2. Memory 应该放在 System Prompt 的哪个位置？

答案其实是**最前面**。

为什么？因为 prompt 变化少的那种放在前面，缓存命中率高，可以降成本。如果说你放在最后是因为“模型对最近的内容更敏感”，那是把因果关系搞反了。

### 3. Skills 和 MCP 分别是什么关系？

Skills 和 MCP 本身不是一个层次的东西。

## 最重要的是工程化

最重要的是工程化，这是很多人低估的。

如果前面的问题问完面试还有时间，他会问你工程实践——这是很多做 Agent 的人翻得最惨的地方。比如说：上下文管理怎么分层的？每层什么时候触发 [unclear audio: 噪音]、怎么处理？Precation 怎么设计？

你答不出来，那就告诉面试官你根本没有做工程化。

### 评测（Eval）

还有一个非常重要的就是 **eval**。很多人简历上要么就没数据，要么就是有准确率，比如说 90% 几。

面试官一问：测试集怎么构建的？覆盖哪些 case？评估指标是什么？怎么自动化回归？这很多人答不上来。

如果这个项目没有做过 [unclear audio: 医保] 评测的话，那基本上就不是上线的东西了。哪怕你项目简历很小，你都要加这个东西。哪怕你手动测了 50 个 case，也非常有价值。

## 软实力考验

最后还有一块软实力的考验。他们可能会问你一些学习方面的东西：平时怎么写代码、看哪些信息源？

这个东西没有标准答案。但如果你能讲清楚：你是怎么用这个 Agent 的 prompt、MCP 怎么弄的、怎么去搭这个 [unclear audio: dock tree]、让这个 Agent 不重复探索、怎么把框架的最佳实践放到这些 skills 里面、主动构建自己的工作流——说明你不只是在用它了，你是深度在用它。

为什么？

但凡你是深度用 AI 的话，你会发现你有多个 AI 工具，它们之间新开项目、新开页面都没记忆，你都会自己去解决这个记忆问题，你会让所有的工具统一记忆。你还会去搭自己的工作流。只要你深度接触，或者说你正在弄，你都会想办法弄的。如果只是用一用，那说明你其实很浅。

还有就是信息源。这个行业变化是以周为单位的。如果你每天能说出自己在刷 **Hacker News**、关注 **YC** 和 **OpenAI** 员工在 X 上面的发言，看前沿 blog——面试官会觉得你在这个领域的参与度是不一样的。

为什么？因为你能看这些东西，说明一定是遇到问题才会看。

## 总结

最后说一句：进这个方向，面试核心逻辑已经变了。

你的项目一定要经得起深挖，深挖就是要有东西。被深挖的前提：

1. **简历写得过关**；
2. **面试时把简历中的每一条内容都整理好了**，随便问、随便说你都整理得出来。

面试官直接问你，包括追问你都能答，因为你提前准备过。这是很重要的一个点。
