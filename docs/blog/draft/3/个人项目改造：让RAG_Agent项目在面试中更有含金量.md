# 个人项目改造：让 RAG/Agent 项目在面试中更有含金量

## 问题：个人项目写得太单薄，怎么升级？

很多同学找实习时候都会遇到：项目看起来太简单，面试官没什么可问的，整个面试几分钟就结束了，也展现不出你的能力。

这篇文章分享一个真实案例：28 届硕士同学，有充足时间准备，如何把一个简单的 Agent 编排项目改造成让面试官追着问的高分项目。

## 改造思路：从单 Agent 到多 Agent 系统

原来的项目只是一个简单的 Agent 工具编排，参考 LangChain 示例写的。改造方向：对标腾讯 **Worker**、**Marvin** 这类产品，升级成**多 Agent 协作系统**。

### 为什么这么改？

- Marvin 这类项目里，有专门处理文件的 Agent，有专门 coding 的 Agent，分工明确
- 单 Agent 项目内容太少，面试官没什么好问的；多 Agent 项目包含了上下文管理、记忆系统、任务分工，可聊的点一下子就丰富了
- 本质上就像一个"数字员工"：你下发指令，有好几个 Agent 帮你分工干活

### 具体升级方向

1. **从网页端改桌面端**：只有桌面端你才能接入系统级操作、操纵电脑，这才是真正能干活的 AI  Agent
2. **增加系统级能力**：自净化、自升级、自记忆、自我迭代，这些都是很好的亮点
3. **不要用 LangChain，手搓核心逻辑**：个人项目要想有含金量，尽量手搓，不要全靠大框架拼积木。面试官一问框架底层你就懵了，手搓更能体现你的功底
4. **尽量轻量级框架**：如果一定要用框架，尽量找轻量级的
5. **可以增加的亮点功能**：
   - 迭代驱动、可观测性
   - 评测闭环
   - 多 Agent 编排
   - 手机发消息，电脑自动干活（类似 Open Sora 的思路）

改造完之后，这个项目比单纯一个 Agent 工作流、单纯一个 RAG 项目有价值得多。

## 最快拿到实习：补一份真实的实习经历背书

第一份实习最难找，本质上是个死循环：
- 公司要你有实习经验才招你
- 你没实习经验所以找不到实习

破解方法：**给你的 RAG 知识库项目补上一段真实的实习经历**。

操作思路：
1. 找一个你想去的一线城市，找一家规模不大不小的真实公司
2. 匹配一个真实存在的业务场景
3. 把项目背景、你做的工作、数据成果都梳理清楚
4. 提前把面试官可能问到的问题都准备好：
   - 项目整体怎么做的
   - 具体某个功能（比如 Memory 记忆模块）你是怎么实现的
   - 遇到了什么问题，怎么解决的

就算你第一场面试发挥不好，多面几次迭代一下，慢慢就非常有经验了。而且很多面试官其实也不是特别懂，只要你能把整个流程说得清清楚楚，就能过关。

## 总结：不同时间，不同策略

### 如果你时间充足（提前大半年准备）
- 个人项目一定要好好迭代
- 从单 Agent → 多 Agent 系统升级，做能真正干活的系统级 Agent
- 尽量手搓核心逻辑，不要全靠 LangChain 拼积木

### 如果你马上秋招，没时间慢慢做项目
- 直接给简历补上企业项目/实习经验
- 把项目背景、过程、成果都梳理得清清楚楚
- 提前准备好面试官可能问到的问题

按照这个思路改造完，你的简历既有公司实习背景，又有一个能体现技术功底的多 Agent 个人项目，比单纯放一个假的 RAG 项目有价值得多。
