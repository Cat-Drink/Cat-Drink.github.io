# AI技术与产品能力总结

本文档整理了文件夹中所有图片的内容，涵盖AI项目初始化方法、AI产品经理能力要求、AI开发理念等主题。

## 一、AI项目初始化的三步法（提示词大都督）

### 第一步：开启全局上下文并做全盘扫描

在新打开一个项目时，不要上来就让AI写代码。你需要先让它做一次"全局扫描"，建立对你整个工程的"心智模型"。特别是在使用现代AI IDE（像Cursor或是带Agent的编辑器）时，这步极其关键。

**建议提问：**
"你现在是一个资深的前端架构师。我已经将整个代码库作为上下文发给了你。请你先不要修改任何代码，只做仔细的阅读和审查。
请用中文为我总结这个项目的核心信息：
1. 这个项目使用的核心技术栈是什么？（包括框架、UI库、状态管理、构建工具等）
2. 请梳理并解释目前的目录结构（如src/下各个子目录的作用）。
3. 项目的入口文件和核心路由体系是如何组织的？"

### 第二步：深挖工程规范与约定（极其重要）

等它梳理完骨架后，你需要让它提取项目里的"潜规则"。这能保证后续它生成的代码风格与现有代码完全一致，而不是自编自导。

**建议提问：**
"非常好。现在请深入到具体的业务代码层面，帮我总结一下目前项目中的编码约定：
1. 数据请求：项目里是如何封装和发起后端API请求的？（是否有统一的axios拦截器？如何处理跨域和Token？）
2. 状态管理：全局状态（如用户信息、配置）存储在哪里？采用了什么模式？
3. 样式与组件：项目主要是用什么方式写样式（如Tailwind还是普通的CSS/SCSS）？通用UI组件和业务组件是如何划分的？
4. 有没有我需要特别注意的特殊逻辑或潜在的技术债？"

### 第三步：生成"记忆面包"（沉淀文档）

当AI完美回答了前两步后，它此时的脑子里包含了这个项目最完整的信息。但只要开启新对话，它就会遗忘。所以，你需要让它把这些信息固化下来。

**建议提问：**
"基于你刚才的审查结果，请帮我生成一份详尽的PROJECT_ARCHITECTURE.md文档。
这份文档的目的是：以后当我开启新的AI会话时，只要让AI读取这个文档，就能立刻了解项目的全局规范和架构。
请排版清晰，包含技术栈概览、目录说明、核心开发规范（如怎么加新页面、怎么调接口）等关键信息。"

**效率小贴士：**
拿到生成的PROJECT_ARCHITECTURE.md后，把它保存在项目根目录下的doc/文件夹里。以后每次你想开发新功能，开启一个新的AI对话窗口，第一句话就是："请先阅读@doc/PROJECT_ARCHITECTURE.md，然后帮我实现xxx功能。"这样写出来的代码质量会高得惊人。

## 二、AI产品经理的核心能力要求（洋言洋语说）

### 1. 对AI技术发展保持敏锐嗅觉

这一条几乎出现在所有招聘JD里。不是让你去训模型，而是你得知道：什么模型能干什么、不能干什么、边界在哪里。

GPT、Claude、DeepSeek、Gemini这些主流模型的能力差异，RAG、多模态、MCP这些概念，不需要精通，但要能说清楚。

**职位描述示例：**
- 负责AI应用的产品规划、设计与落地（效率工具、智能助手、自动化流程等）；
- 深入理解团队（产品/运营/研发/数据等）的真实工作场景，把AI嵌入到日常工作流中；
- 将零散的AI使用方式，产品化、系统化，沉淀为可复制的内部能力，提升个人和组织的产出；
- 影响和带动团队更高质量地使用AI。

**任职要求示例：**
- 有很强的自驱力和学习能力，在自己的工作中高频、深度使用AI；
- 具备产品思维，能把"一个好用的Prompt/Workflow"升级为"一个可用的产品"。

### 2. Prompt工程 + Agent/工作流设计

这是当下AI PM最硬核的技能，不只是会写Prompt。而是能把业务场景拆解成完整链路：输入 => 任务规划 => 工具调用 => 结果校验 => 异常兜底。

能设计Agent行为规则、编排多步工作流的人，非常加分。

### 3. 具备AI Coding能力

不是让你当程序员，而是能用Cursor、Claude Code这类工具快速搭Demo验证想法。

"能独立完成简单AI工具开发/调试"，这句话已经直接写在JD里了。

### 4. 有AI产品从0到1的落地经验

公司希望你有现成的AI产品经验，最好带行业垂类背景，入职就能上手。所以需要你具备完整的全流程经验：产品立项、市场调研、需求分析、方案设计、研发跟进、推动上线。

核心是能讲清楚：为什么做、怎么做、效果如何、踩过什么坑。

### 5. 有可展示的个人AI作品

没有AI产品经验也不要紧，这一点可以帮你补足。自己vibe coding一个AI项目，或者在自媒体平台持续输出：AI行业洞察、AI产品测评、使用体验分析，都可以。

作品集就是最好的简历。

## 三、AI开发的理念转变（释然_AI）

### 心智转换：从"开发Agent"到开发Harness

当一个人说"我在开发Agent"时，他只可能是两个意思之一：

1. **训练模型**：通过强化学习、微调、RLHF或其他基于梯度的方法调整模型，用它们来塑造模型的行为。这是DeepMind、OpenAI、腾讯AI Lab等机构的工作。

2. **构建Harness**：编写代码，为模型提供一个可操作的环境。这是我们大多数人的工作。

### Harness的构成

Harness是agent在特定领域工作所需要的一切：

```
Harness = Tools + Knowledge + Observation + Action Interface + Permissions
```

- **Tools**：文件读写、Shell、网络、数据库、浏览器
- **Knowledge**：产品文档、领域资料、API规范、风格指南
- **Observation**：git diff、错误日志、浏览器状态、传感器数据
- **Action**：CLI命令、API调用、UI交互
- **Permissions**：沙箱隔离、审批流程、信任边界
