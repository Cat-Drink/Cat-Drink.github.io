# 通过状态机和约束，让AI不再瞎跑

相信大家在使用AI编程工具的时候都有过这种体验：

你跟AI说："帮我在桌面写个报告，要PDF格式的。"

结果：
- 它开始帮你写文档，然后又帮你写程序
- 又是格式转化，又是调试代码错误
- 陷入到各种工具的调用循环中
- 保不齐还会突然跑偏，扯到其他话题上
- 最后一看，token烧了快上千个还没写完

这种情况本质上就是AI工具在设计的时候没有把"方向盘"做好，导致模型想到哪说到哪，没有约束。

## Stream Code 的解决方案：状态机 + 守卫

简单点说，就是给AI装了一条有红绿灯的单行道。AI一旦走上这条路，就要历经八个阶段，**只能往前走，不能跳、不能倒**。

### 八个固定阶段

1. **理解需求**
2. **探索代码**
3. **设计方案**
4. **做规划**
5. **写代码**
6. **验证**
7. **交付**
8. **关闭**

### Guard 守卫机制

如果AI想偷懒，比如直接从设计跳到编码，会有一个叫 **Guard** 的守卫直接把它挡回去：**没做规划不能过**。

而且这个拒绝动作也会被记录下来，**日志只能追加不能修改**，跟黑匣子的逻辑差不多，方便追溯问题。

## 工作流支持：自定义流程，自动退出

除了状态机，每个任务还可以定义工作流（Workflow）：

比如测试驱动开发（TDD）：
- 先写测试让它红
- 再写最少代码让它绿
- 最后重构

AI不需要你每一步盯着催，它会自己按节奏走，走完了工作流自动退出，**不占用后续token**。

## 效果

引入状态机和约束之后：

1. **任务闭环率大幅提高**：AI不再跟你东拉西扯，每一步都有规矩，每一个跳转都记录在案，大幅提高了最终结果的完成度

2. **更快更省**：原来要花十分钟搞定的任务，现在可能只要五分钟就搞定了，而且质量更高，token消耗更低
