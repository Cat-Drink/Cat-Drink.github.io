# 5分钟快速弄懂RAG核心要点

RAG 是 **检索增强生成（Retrieval-Augmented Generation）** 的缩写，是一种扩展大模型能力的技术方案。

## 为什么需要RAG？

大模型虽然强大，但并不完美，存在几个关键问题：

1. **知识过时**：模型的知识取决于训练数据，2025年训练的模型只知道2025年之前发生的事情，之后发生的新事件它不知道
2. **私有数据缺失**：未公开的数据（比如企业内部信息、个人整理的资料）它不知道
3. **幻觉问题**：不知道信息的时候，它还会"胡说八道"编造答案

那为什么不直接用其他方案解决？

- **方案1：不断重新训练模型** → 硬件成本和时间成本都太高，一般企业和个人折腾不起
- **方案2：提问前把所有文档都给大模型** → 大模型对上下文长度有限制，输入太多它就找不到重点，最终答案质量也不理想

针对这些问题，**RAG应运而生**，它的核心思想很简单：**先查资料，再回答问题**。

## RAG的工作流程

RAG主要分为两个阶段：**数据准备阶段**和**推理使用阶段**。

### 第一阶段：数据准备（好比建图书馆）

就像提前建好图书馆把书准备好，步骤如下：

1. **收集文档**：收集相关的业务数据文档，包括PDF、Word等各种格式
2. **文档切块**：把大文档切成小块文本
3. **向量化（Embedding）**：将每一段文本转换成一组数字（向量）
4. **存储到向量库**：把转换后的向量和原始文本都存储到向量数据库中

> 为什么要用向量数据库？因为它可以**快速检索**，能迅速找到相关内容。

### 第二阶段：推理使用（用一个例子说明）

假设用户问：**"iPhone最新款用的是什么芯片？"**

流程是这样的：

1. **问题向量化**：把用户的问题转换成向量
   > ⚠️ 重要：这里用的向量化模型必须和数据准备阶段用的是同一个，否则转换出来的向量空间不对，结果会很差

2. **相似度匹配检索**：拿着问题向量去向量库中匹配，找出相似度最高的前K条结果
   - K可以根据业务需求设定，比如取前10个或前20个

3. **组装提示词**：把检索到的文本作为参考资料放在前面，用户问题拼在后面，一起传给大模型

4. **大模型回答**：大模型根据参考资料来回答问题

## 向量是如何匹配的？

向量匹配的核心是**计算向量相似度**：判断两个向量之间的距离，**越相似的向量距离越近**。

举个直观理解：
- 在二维坐标系中，数据库中已有的数据分布在不同位置
- 比如"苹果最新款信息"在某个位置
- 用户问题转换后的向量也会落在向量空间的某个位置
- 如果两个位置离得很近，就认为它们相似，就把这条数据匹配出来

实际应用中不是二维，而是**多维数字空间**，原理是一样的。

## RAG本质总结

RAG的本质其实很简单：**给大模型一本参考书，让大模型根据参考资料进行回答**。

这样既解决了知识过时、私有数据的问题，又避免了全量重新训练的高成本，同时也不会超出大模型的上下文限制。
