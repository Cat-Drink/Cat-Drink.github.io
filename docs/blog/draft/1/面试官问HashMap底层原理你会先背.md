# 面试官问 HashMap 底层原理，你会先背？

## 用生活场景理解 HashMap

**HashMap 底层 = 数组 + 链表 + 红黑树**

### 核心类比：城中村快递架

- **数组**：一排格子（快递架）
- **每个包裹**：一个 Key-Value
- **放包裹的位置**：看单号（Key hashCode），用 `hash & (table.length - 1)` 算该放哪个格子

### 哈希冲突
- 两个包裹算到同一个格子 → 用链表串起来（糖葫芦式）
- 链表短的时候好找，太长就要从头摸到尾，查找效率从 O(1) 降到 O(n)

### 链表 → 红黑树（JDK 8+）
- 同一个桶节点超过 **8 个**，且数组长度至少 **64**
- 链表换成红黑树，查找从 O(n) 变成 **O(log n)**
- 节点减少到 **6 个**时退回链表，避免来回折腾

### 扩容（Resize）
- 默认负载因子 **0.75**
- 格子用到约 75% 就买两倍大的新架子
- 所有包裹重新算位置搬过去（Rehash）
- 扩容很耗性能，如果知道要放很多数据，一开始就把容量估大

### 并发场景
- **HashMap 不是线程安全的**
- 多线程下可能数据覆盖、丢数据
- 老版本并发扩容还可能把链表扳成环
- **并发场景用 ConcurrentHashMap**

## 面试官真正想问的

> 不是要你背"数组 + 链表 + 红黑树"的定义，而是要知道：
> - 东西怎么放
> - 撞了怎么排
> - 架子满了怎么搬
