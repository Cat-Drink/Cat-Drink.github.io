# 想毁掉一个人，就让他不停地 Vibe Coding

## 核心观点

> "让非技术人员写代码"是 Vibe Coding 布下的最大的骗局。

## 什么场景适合 Vibe Coding？

- 没有工程化思维的情况下
- 写个记账本、番茄钟玩玩——完全没问题
- **但这纯属自嗨，要想具备商业价值还远远不够**

## 初级程序员 vs 架构师

| 角色 | 能力 | 薪资 |
|------|------|------|
| 初级程序员 | 纯敲代码 | 低 |
| 架构师 | 把需求设计成框架 | 高 |

核心差距在于**架构思维**。

## Vibe Coding 的陷阱

- 动动嘴就能生成一个网页的即时奖励感太强
- 容易让人沉迷于"伪生产力"

## 让 Vibe Coding 成为真正生产力的前提

开始 Vibe Coding 前，需要先搞懂四个问题：

1. **核心链路是什么？**
2. **每个卡点怎么验证？**
3. **能不能优先用文档？文档里有没有成功标准？**
4. **如何将 Vibe Coding 转换成真正的生产力？**
