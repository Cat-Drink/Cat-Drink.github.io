1、开发更看重候选人是否经过系统性的专业训练
2、多数求职者只会盯着开发岗投递，实际上**测试、数据分析、技术支持、实施类岗位**市场需求量同样很大，准入门槛也更贴合普通学生
3、拆解岗位 JD，**自身条件满足一半以上再投递**，避免无意义消耗时间
4、投递前一定要仔细阅读 JD 中的岗位职责描述，判断日常工作内容是否是自己可以长期接受的。
5、采用"先中小厂练手，后冲刺大厂"的投递顺序
6、简历遵循"少而精"原则，重点打磨项目经历，同时一定要增加量化成果描述，哪怕数据规模不大，也能让项目脱离"课程作业"的单薄感，更有真实落地感。
7、面试前反复打磨项目讲解，用固定结构化话术练习.例如可以固定按照「**项目背景 → 业务目标 → 个人职责 → 技术实现 → 落地成果**」的框架梳理项目，反复口述练习。面试时讲解不会卡顿，自信心更强。
8、求职是持续筛选的过程，单次失败不能定义全部结果
9、可以对意向公司 / 收到面试邀约前置调研

### 自我介绍规范
禁止照着简历原文朗读，使用三段式结构，时长控制在 1 分 30 秒以内：
过去做了什么 \+ 现在为什么应聘本岗位 \+ 入职后能为公司带来什么；
交谈时目光落在面试官眉心位置，兼顾自信与舒适，避免直视对视产生尴尬。

### 面试必问清楚 3 项核心权益
无需碍于情面，均属于求职者合法知情权：
1. 薪资核算：底薪税前 / 税后标准、绩效发放规则、是否存在年终奖；
2. 五险一金：社保公积金缴纳起始月份、缴费基数、公积金缴纳比例；
3. 工作节奏：加班频次、双休 / 大小周制度、是否发放加班费。

### 面试结尾反问环节技巧
HR 收尾询问「你还有什么问题吗」时，不能直接回答没有，会让面试官认为你对岗位缺乏兴趣；
提前准备 1\-2 个问题，示例：
1. 新人入职是否配备专属带教导师；
2. 该岗位日常以团队协作为主，还是独立完成工作居多；


在自我介绍中融入对项目经历的介绍，埋下伏笔给面试官进行深挖，如果面试官接着问就是有戏，如果顾左右而言它就是没戏了，而且一定要讲到面试官喜欢听，或者说JD符合的关键点，讲到面试官心里才行

面试的时候容易大脑空白语无伦次，可以调整心态，当成平等交流而不是一种考核，讲话的时候用短句框架，放慢语速
，把高频问题的答案写成**短句**，默念熟悉框架即可。要稳。把每次面试都当经验积累，锻炼自己的面试能力


面试的表达要具有逻辑性，不要啰嗦，避免脑子空白，要组织好语言，抓住面试官想了解的核心
可以先说结论，再说为什么，怎么做，并加上支撑案例，讲话的时候可以多加连接词进行关联，这样表达更加条理清晰。也可以提前预判高频问题，给每个问题预设逻辑框架
常见问题框架参考：
- **自我介绍/项目经历**：用 **STAR 法则**（情境Situation → 任务Task → 行动Action → 结果Result）
- **优缺点问题**：优点 = 结合岗位需求 + 案例；缺点 = 说非核心缺点 + 改进措施
