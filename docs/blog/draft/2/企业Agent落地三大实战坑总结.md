# 企业Agent落地三大实战坑总结

## 一、第一大坑：知识库失控，先乱知识库再谈模型工具

上个月帮一个 300 人左右的公司做 Agent 落地，一开始我以为最难的是模型和工具。
结果跑了一周才发现，最先翻车的不是模型，而是知识库。

大家刚开始都很兴奋，什么资料都想往里塞。
文档重复、目录混乱、索引失效，
Agent 看起来接入了知识库，实际上已经找不到重点了。

这个坑本质上不是员工不会用 AI。
而是新手刚接触 Agent 时，很容易把知识库当资料仓库。
什么都想沉淀，什么都不舍得删。
短期看是在积累，
一周后就会变成垃圾上下文。

所以企业落地 Agent，第一步不是上工具。
而是先定知识库规则。
目录怎么分，内容怎么写，什么要删，什么能留。
工程里可以理解成：
先做知识库 lint，再谈规模化使用。

## 二、第二大坑：试点人群选错，优先一线而非中层

第二个坑，是试点人群选错了。
一开始只给中层配了订阅。
逻辑看起来没问题：他们懂业务，也更像能推动事情的人。
但真实跑下来，中层未必最有动力。
有活可以分下去，有问题可以让别人处理，
让他重新教 Agent，反而更麻烦。

真正适合试点的，往往是一线员工。
因为他们每天被重复工作卡住，
有场景，有痛点，也最需要工具帮他们省时间。

## 三、第三大坑：规划周期过长，推行月度滚动迭代

第三个坑，是规划做太长。
刚开始我们也画过半年规划。
但跑了一个月后发现，很多优先级已经变了。
有些事原本以为要一个月，三天就有结果。
有些事原本以为很简单，后来发现外部依赖一堆。

所以我现在更相信一个月滚动：
先定规则，再找一线场景，
每周 review，随时调整方向。

Agent 落地不是把工具发下去。
而是把知识、权限、流程和真实痛点，重新接进一个能跑的工作流里。
