# 远程仓库

## git和代码托管中心

我们的代码不能总是放在本地,因为总是放在本地,一旦电脑出现故障,数据将丢失,怎么共享呢,这里我们需要一个服务器, 我们可以把代码放到服务器上,然后让别人下载,这样我峨嵋你既可以备份代码,也可以进行团队协作开发

代码托管中心是基于网络服务器的**远程代码仓库**，一般简称为远程库。

除了自己搭建服务器,其实我们可以使用一些免费的远程仓库,远程仓库有很多,常见的免费互联网远程仓库托管服务如下:

[github  是一个基于git实现在线代码托管的仓库，向互联网开放，企业版要收钱（外网）](https://github.com/)

[gitee    即码云，是 oschina 免费给企业用的，不用自己搭建环境（国内网站）](https://gitee.com/)

[gitlab   （局域网）类似 github，一般用于在企业内搭建git私服，要自己搭环境](https://about.gitlab.com/)

- GitHub(gitee)、GitLab 不同点：

1. GitHub如果使用私有仓库是需要付费的，(2019年开始私有仓库也是免费的但是只能3个人协同开发,想要更多需要收费)，GitLab可以在上面搭建私人的免费仓库。
2. GitLab让开发团队对他们的代码仓库拥有更多的控制，相对于GitHub，它有不少的特色：
   1. 允许免费设置仓库权限
   2. 允许用户选择分享一个project的部分代码
   3. 允许用户设置project的获取权限，进一步提升安全性
   4. 可以设置获取到团队整体的改进进度
   5. 通过innersourcing让不在权限范围内的人访问不到该资源

### git团队协作机制

#### 团队内协作


一个本地库将代码推送push到代码托管中心，另一个本地库复制代码下来（clone)进行修改后再推送到代码托管中心，原来的本地库再拉取(pull)提交上来的代码到本地库
github授予权限，链接邀请函地址，打开邀请函即可。

#### 跨团队协作

这里运用到两个远程库之间的交互


`fork`将其他远程库的代码在代码托管中心上复制一份到该远程库

该远程库对代码修改且推送后，发出拉取请求`pull request`，再通过审核，另一远程库再(拉取)`merge`过来

1. 搜索项目或访问项目链接
2. fork插入该文件到该远程库
3. 提出拉取申请pull request
4. 同意申请

### SSH免密登录

实际上git 不仅仅支持用户名密码方式的配置,可以有另外一种相对更加安全的配置即ssh 方式配置

ssh 方式的底层原理

ssh连接底层是RAS加密算法,又称非对称加密,是一种现在公认的最安全的加密方式

数学基础好的同学可以研究一下https://www.cnblogs.com/cjm123/p/8243424.html

## 远程仓库操作命令

##### git remote                                                 查看远程  列出指定的每一个远程服务器的简写

###### git remote rm 远程仓库别名                        移除远程仓库和本地仓库的关系(只是从本地移除远程仓库的关联关系，并不会真正影响到远程仓库)

###### git remote show  <仓库简称>                        查看远程仓库详细地址

###### git remote -v                                                查看当前所有远程地址别名


###### git remote add 远程仓库地址                        增加远程仓库

###### git remote add 别名 远程仓库地址                给远程仓库起别名(在添加远程仓库的时候)



##### git push 别名(或网址） 本地分支名                推送本地分支上的内容到远程仓库

`git push origin master`将本地仓库该分支推送至远程仓库的该分支



推送后会出现凭据，可在Windows凭据管理器查询



###### git push [-f] [--set-upstream] [远端名称 [本地分支名][:远端分支名] ]

如果远程分支名和本地分支名称相同，则可以只写本地分支`git push origin master`

`-f `表示强制覆盖

`--set-upstream `推送到远端的同时并且建立起和远端分支的关联关系。

```
git push --set-upstream origin master
```

如果当前分支已经和远端分支关联，则可以省略分支名和远端名。

git push 将master分支推送到已关联的远端分支。

###### git push --force 远程仓库别名 远程分支名        强制推送与远程版本有差异的本地版本

```
git push --force origin master
```

###### git push 远程仓库别名 --delete 远程分支名        删除主机的分支可以使用`--delete`参数

命令表示删除 origin 主机的 master 分支：`git push origin --delete master`

###### git push -u 远程仓库别名 分支名                        设置默认推送分支

加了参数-u后，以后即可直接用`git push`代替`git push origin master`意思就是我们不用在后面加 origin + 要提交的分支名 ，直接 git push 即可。但是前提是，第一次提交需要加 -u参数，以后的提交就直接可以 `git push`

##### git clone 远程仓库地址                                将远程仓库的内容克隆到本地



在没设置别名前，网址默认是origin别名



##### git origin 远程地址                                         将该远程仓库设为origin

##### git fetch  <shortname>  <分支名称>                从远程仓库拉取 (拉取到.git 目录,不会合并到工作区,工作区发生变化)

##### git pull 远程仓库地址别名 远程仓库分支名        拉取远程仓库最新内容且合并本地内容

从远程仓库拉取 (拉取到.git 目录,合并到工作区,工作区不发生变化) = fetch+merge

将远程仓库对于分支最新内容拉取下来后与当前本地分支直接合并



###### git pull  远程仓库地址别名 远程仓库分支名  --allow-unrelated-histories  强制拉取合并

注意：如果当前本地仓库不是从远程仓库克隆，而是本地创建的仓库，并且仓库中存在文件，此时再从远程仓库拉取文件的时候会报错（fatal: refusing to merge unrelated histories ），解决此问题可以在git pull命令后加入参数`--allow-unrelated-histories` (如上 命令)

##### git remove 远程仓库别名                                删除远程仓库别名

##### 下载

源代码是发布在GitHub上的，但是当我们无法正常访问GitHub的时候可以先下载发布在[gitee上的旧版本](https://gitee.com/jiulang/fast-github/releases) （已失效），等能正常访问GitHub的时候再下载使用[最新的版本](https://github.com/dotnetcore/fastgithub/releases)。目前还是2022年3月份更新的版本

https://github.com/dotnetcore/FastGithub

##### 安装及使用

下载下来的是个zip的压缩包，解压缩之后找到.exe的运行程序，双击运行程序

然后以管理员的身份运行cmd，输入运行FastGithub的命令即可运行 D:\software\fastgithub_win-x64是你将fast GitHub下载存放的位置

```SQL
D:\software\fastgithub_win-x64\FastGithub.exe start    #启动fast GitHub

D:\software\fastgithub_win-x64\FastGithub.exe stop      #停止fast GitHub
```

启动之后fast GitHub会自动更改你的dns

- 在每次访问GitHub之前记得要先保证FastGithub在运行的状态

在安装完成之后可以直接点开fastgithub的执行文件就可以运行程序，顺利连接到Github。如果不能正常打开.exe的执行文件的话还是使用命令去启动，刚开始直接将执行文件拖到桌面是不能使用的，是在文件夹里面打开之后固定在任务栏之后可以正常使用。

1. 当GitHub能正常跳转之后我们就可以下载安装GitHub上的[FastGitHub的最新版本](https://github.com/dotnetcore/fastgithub/releases)，或者现在的[fastgithub官网](http://dedeapps.com/dotnetcore/FastGithub/download)上也是有最新版的下载链接的，安装使用方法一致
2. Linux下FastGithub及使用：需要下载linux上的FastGithub来进行加速

FastGithub本地代理使用的都是“公用电信网提供的国际出入口信道”，从国外Github服务器到国内用户电脑上FastGithub程序的流量，使用的是正常流量通道，其间未对流量进行任何额外加密
