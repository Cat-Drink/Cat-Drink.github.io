# Trae 实用技巧汇总

## 第一部分：隐藏指令人格类功能

### 1. /roast 犀利点评指令

#### 适用场景

不想听客套话术，需要直白指出方案 / 想法漏洞，或需要轻松方式挖掘创意问题点时使用。

#### 核心操作

输入指令：`/roast [话题]`

#### 执行效果

切换「犀利吐槽」人格，抛弃温和客套风格，以幽默讽刺的口吻直击内容痛点，不迎合、不委婉，不直接落地执行，优先吐槽审视你的思路。

#### 配套灵魂拷问校验清单（自查方案漏洞）

1. 你解决什么问题？——"呃… 就是… 赚钱的问题？"

2. 目标用户是谁？——"所有人！"

3. 和竞品有什么不同？——"我们更… 新？"

4. 怎么推广？——"发朋友圈？"

5. 预算多少？——"先做着看…"

#### 经典失败案例参考

- "社交 + 电商 + 直播 + 游戏" 超级 APP：落地难度极高，几乎无法平衡资源

- "中国版 XXX"：对标产品自身尚未实现盈利，赛道无成熟验证

- "绝妙想法，只差程序员"：脱离技术落地可行性，开发成本与风险完全缺失考量

- "先做 MVP 快速试错"：无迭代规划，反复试错造成大量时间成本浪费

### 2. /rap 说唱歌手创意指令

#### 适用场景

内容创作、活动策划、灵感枯竭，需要新颖创意表达形式时。

#### 核心操作

输入指令：`/rap [话题]`

#### 执行效果

切换「说唱歌手」人格，输出结构完整的中文 Rap 歌词，标准结构包含：Verse、Hook、Bridge、Outro；要求押韵紧凑、节奏流畅、态度鲜明。

#### 能力价值

TRAE 不局限于办公、方案类正经任务，可作为创意表达辅助工具，产出趣味化、有记忆点的内容。

### 3. /debate 多方辩论思考指令

#### 适用场景

撰写方案、梳理观点、学习陌生主题，思路模糊、无法多角度分析问题时。

#### 核心操作

输入指令：`/debate [议题] --sides=3`

#### 执行效果

自动生成 3 个完全对立的立场，开展完整辩论，输出结构化 Markdown 辩论文档，打破单一视角局限，实现全面思考。

#### 输出文档结构

1. 辩论基础信息：辩题、三方立场关键词

2. 开篇立论：三方分别阐述核心观点

3. 交叉质询、自由辩论环节

4. 总结陈词：横向对比各方优劣势、适配人群、风险、底层理念

#### 示例议题

年轻人毕业后第一选择：先就业、先创业，还是先深造？

- A 方（先就业）：务实、经济独立、积累实操经验

- B 方（先创业）：激情、抓住时代机遇、全方位成长

- C 方（先深造）：高学历起点、拓宽未来选择

## 第二部分：Trae + 飞书 CLI 自动化联动功能

### 1. 自动化信息图谱搭建（行业资讯定时沉淀）

#### 适用场景

需要每日跟踪行业行情、价格波动、竞品动态、知识库更新，手动整理信息效率低下人群。

#### 核心操作

1. 完成飞书 CLI 授权接入，创建定时自动化任务

2. 配置规则：TRAE 每日自动执行信息采集，结果同步至飞书文档

#### 核心价值

区别于单次问答，实现**每日自动重复执行**，长期沉淀行业信息资产，形成专属信息图谱。

#### 实操案例

设置定时任务：每日自动采集国际金价、全国 92/95 号汽油价格，同步生成标准化数据表格至飞书云文档，留存每日价格走势记录。

### 2. 直播 / 会议纪要一键精炼发布

#### 适用场景

直播、分享会、线下会议、访谈内容整理，录音文字稿冗长、口语化严重，需要快速产出可对外发布的精简内容。

#### 核心操作

1. 授权飞书 CLI，读取飞书妙记转录的会议文字内容

2. 使用标准化 Prompt 交给 TRAE 处理，自动生成规整文档并同步至飞书

#### 标准化 Prompt 模板

```Plain Text
请按照 forum-summary-skill 规范，将以下直播纪要精炼为论坛发布版：
要求：
1. 提取核心观点，去除口语化表达
2. 结构化重组：标题、嘉宾、章节、表格、Q&A
3. 字数控制在原文 20-25%
4. 保留关键数据、案例、方法论
5. 生成 Markdown 格式

内容：
[粘贴飞书纪要内容]
```

#### 核心价值

打通 TRAE 内容生成与飞书文档归档、发布链路，将重复的文字整理工作标准化，批量处理多期分享内容。

## 第三部分：Trae 结合 MCP 上下文工程

### 一、四大核心技巧

#### 1. 写入（持久化上下文，跨会话复用）

将用户偏好、项目配置、核心结论等高价值信息持久化，突破单次对话限制。

**操作步骤：**
1. 添加存储型 MCP Server（推荐 `Knowledge Graph Memory MCP`），以知识图谱结构化存储上下文
2. 新建自定义智能体，绑定上述 MCP Server
3. 在系统提示词中划定持久化信息类型与存储逻辑

**效果：** 用户提问某框架初始化方式时，智能体自动读取知识图谱中关联的初始化步骤、注意事项，无需重复提问。

---

#### 2. 选取（动态筛选相关上下文，提升准确性）

通过 MCP 调用外部检索工具，动态抓取与当前任务强相关的信息，屏蔽无关干扰。典型搭配：RAG + MCP 对接知识库。

**操作步骤：**
1. 部署 `Sequential Thinking MCP Server`，分步拆解检索逻辑
2. 将 `@builder` 切换为 `@builder with MCP`，加载全部 MCP 服务
3. 选用 Gemini 2.5 Pro 等高能力模型
4. 输入标准化检索提示词，要求先拆解任务再匹配上下文

**效果：** 智能体自动按阶段匹配对应文档（需求→需求文档、设计→技术选型文档），输出逻辑分层清晰。

---

#### 3. 压缩（精简上下文，降低 Token 消耗）

借助 MCP 调用压缩工具，用更少 Token 承载完整核心信息。

**操作步骤：**
1. 同时部署**文本压缩 MCP Server** + `Sequential Thinking MCP Server`
2. 配置压缩策略：先拆解核心信息层级，对超 500 字内容进行压缩，优先保留技术参数、步骤要点、知识关联关系

**效果：** 1000 字文档压缩至 300 字关键要点，核心信息无丢失，大幅削减 Token 开销。

---

#### 4. 隔离（避免信息干扰，提升复杂任务效率）

依托多智能体架构 + MCP 资源隔离配置，各子智能体只处理自身业务域信息，解决跨域上下文互相干扰问题。

**操作步骤：**
1. 为每个业务模块搭建专属知识库 MCP Server，资源完全独立
2. 创建子智能体，各绑定专属 MCP，仅处理本模块查询，只输出核心结论
3. 配置总控主智能体，按模块分发任务、汇总结论统一输出

**效果：** 多模块接口对接场景下，各子智能体仅返回本模块参数与适配要点，主智能体整合输出完整方案，彻底隔绝冗余信息干扰。

---

### 二、进阶优化建议

#### MCP Server 选型

| MCP 服务 | 适用场景 |
|---|---|
| Sequential Thinking MCP | 复杂任务分步拆解、上下文有序分层管理 |
| Knowledge Graph Memory MCP | 结构化信息存储、知识关联图谱搭建 |

#### 提示词优化

在系统提示词中硬性约束 MCP 调用规范，例如：
- 调用 Sequential Thinking MCP 必须明确任务拆解层级
- 调用知识图谱 MCP 必须提前定义信息关联规则
- MCP 调用失败时自动提示用户检查 Server 配置

#### 日志监控

进入 TRAE MCP 设置页面查看调用日志，分析上下文选取准确率、压缩效果，反向迭代优化 MCP 参数与提示词。

---

### 三、常见问题排查

| 问题 | 排查方向 |
|---|---|
| **MCP 调用失败** | 检查网络连通性、API Key/Token 是否正确；Local 类型检查 NPX/UVX 运行依赖 |
| **上下文选取不精准** | 修改提示词限定检索关键词范围；替换精度更高的第三方 MCP 检索工具 |
| **Token 消耗过高** | 强化压缩规则；收紧选取逻辑，过滤低相关度上下文 |

## 总结

### 功能分类

1. **创意发散工具**：`/roast` 自查方案漏洞、`/rap` 产出趣味创意文案、`/debate` 多角度辩证思考，覆盖思路校验、内容创作、逻辑分析三类需求；

2. **办公自动化工具**：联动飞书 CLI 实现定时信息采集、会议纪要批量精炼，打通数据采集 - 内容处理 - 文档归档全流程，解放重复性整理工作；

3. **上下文工程工具**：通过 MCP 实现写入、选取、压缩、隔离四大核心技巧，有效管理持久化上下文，提升复杂任务处理效率，降低 Token 消耗。

### 组合工作流示例

先用 `/debate` 梳理方案逻辑，再用 `/roast` 排查漏洞，最后通过 MCP 持久化存储结论，结合飞书自动化定时沉淀方案迭代记录，形成完整工作流。
