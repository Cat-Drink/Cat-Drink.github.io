# Linux/shell

## 按修改时间排序查看当前目录文件
ls -lt 按时间升序
ls -lrt 按时间降序

## linux上写一个新文件
- 文本编辑器，如 vi 或 vim 
- echo "xxx" > 新文件名.后缀（ > 是覆盖 ，>> 是追加 ）
- touch 新文件名.后缀

## 查看刚刚的文件并根据文件里第二列内容排序
sort -k2,2 -n -r data.txt
- -n：按数值排序（如果第二列是数字）
- -r：逆序排序

## 主机名映射是哪个文件？
/etc/hosts

## 查看端口号状态命令
- lstat -ant
- netstat -an

## 查看端口占用情况可以使用 lsof 和 netstat 命令
注意：lsof 普通用户可以使用；netstat 普通用户不行，必须是root用户，或 sudo netstat

## 权限765对应的是什么
二进制表示 rwx，r 代表可读，w 代表可写，x 代表可执行。
题目中 权限十进制为 765，
则所有者具有 7 = 111 的权限，即 rwx； 
所在组具有 6 = 110 的权限，即 rw-； 
其他人具有 5 = 101 的权限，即 r-x； 
所以最终权限标识为 rwx rw- r-x。

## Linux中软连接和硬连接的区别？

### 硬链接 (Hard Link)
- **本质**：与原文件指向同一个 inode 的文件
- **特性**：
  - 删除原文件，硬链接文件仍可正常访问
  - 不能跨文件系统创建
  - 不能对目录创建硬链接
  - 修改任意一个链接文件，其他链接文件的内容都会改变
- **命令**：`ln 原文件 硬链接名`

### 软链接 (Symbolic Link / Soft Link)
- **本质**：一个特殊的文件，存储的是原文件的路径（类似于 Windows 的快捷方式）
- **特性**：
  - 删除原文件，软链接会失效（"悬空链接"）
  - 可以跨文件系统创建
  - 可以对目录创建软链接
  - 删除软链接不会影响原文件
- **命令**：`ln -s 原文件/目录 软链接名`

### 主要区别总结
| 特性 | 硬链接 | 软链接 |
|------|--------|--------|
| inode | 与原文件相同 | 不同 |
| 跨文件系统 | 不支持 | 支持 |
| 目录链接 | 不支持 | 支持 |
| 删除原文件 | 仍可访问 | 失效 |
| 大小 | 与原文件相同 | 很小（仅存储路径） |
